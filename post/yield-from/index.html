yield form vs yield

yield from is equivalent to

def func:
	for elem in gen:
		yield elem

where gen is a generator or list

it returns generators in different format. for example

a = (1, 2, 3, 4)

def get(arr):
	yield arr

def get2(arr):
	for x in arr:
		yield x

def get3(arr):
	yield from arr

def println(arr):
	print("type: ", type(arr))
	for x in arr:
		print(x, end=' ')
	print()


println(get(a))
println(get2(a))
println(get3(a))

the output:

type:  <class 'generator'>
(1, 2, 3, 4)
type:  <class 'generator'>
1 2 3 4
type:  <class 'generator'>
1 2 3 4

if the input is list, ie:

b = [5, 6, 7, 8]
println(get(b))
println(get2(b))
println(get3(b))

then,

type:  <class 'generator'>
[4, 5, 6, 7]
type:  <class 'generator'>
4 5 6 7
type:  <class 'generator'>
4 5 6 7

Example

yield version

def nextSquare():
	arr = [1, 10, 20, 50, 100, 150, 1000]

	for x in arr:
		yield x

for num in nextSquare():
	if num > 100:
		break
	print(num)

considering yield => return + keep going. for each nextSquare(), yield gives value i * i to num, then check the num if bigger than 100, print or break. After print or break, go back to operation i += 1 then continues the infinite loop

yield from version

def nextSquare():
	arr = [1, 10, 20, 50, 100, 150, 1000]
	yield from arr

for num in nextSquare():
	if num > 100:
		break
	print(num)

they do the same thing
